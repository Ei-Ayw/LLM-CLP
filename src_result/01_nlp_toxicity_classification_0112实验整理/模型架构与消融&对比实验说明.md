# 模型架构与实验详细说明 (Model Architecture & Experiments)

本文档详细描述了本研究中提出的 **DeBERTa-V3 Multi-Task Learning (MTL)** 模型架构，以及消融实验、对比基线模型的具体设置。

## 1. 模型架构 (Model Architecture)

本研究提出的模型基于 **Microsoft DeBERTa-V3** 预训练语言模型，并结合了多任务学习 (MTL) 和注意力池化 (Attention Pooling) 无缝集成。

### 1.1 骨干网络 (Backbone)
*   **Base Model**: `microsoft/deberta-v3-base`
*   **Hidden Size**: 768
*   **Tokenizer**: DeBERTa-V3 Tokenizer (SentencePiece based)

### 1.2 核心组件 (Core Components)

#### A. 注意力池化 (Attention Pooling)
为了利用序列中所有 token 的语义信息，而非仅依赖 `[CLS]` token，我们设计了一个注意力池化层：
1.  **Score Layer**: 将 Hidden States 映射到标量分数 (`Linear(768 -> 1)`).
2.  **Softmax**: 对序列长度维度的分数进行 Softmax 归一化，得到每个 token 的权重。
3.  **Weighted Sum**: 根据权重对所有 token 的 Hidden States 求加权和。
4.  **Concatenation**: 将 `[CLS]` 向量与 `Pooled Output` 拼接，形成更丰富的特征表示 (`768 + 768 -> 1536`)。

#### B. 多任务学习头 (Multi-Task Heads)
经过共享的投影层 (`Linear -> GELU -> Dropout`) 将维度降至 **512** 后，特征被送入三个并行的任务头：
1.  **Toxicity Head (主任务)**:
    *   结构：`Linear(512 -> 1)`
    *   目标：预测文本是否具有毒性 (Target: `y_tox`)。
    *   损失函数：`BCEWithLogitsLoss`
2.  **Subtype Head (辅助任务 1)**:
    *   结构：`Linear(512 -> 6)`
    *   目标：预测 6 种细粒度毒性类别 (Severe Toxic, Obscene, Threat, Insult, Identity Hate)。
    *   损失函数：`BCEWithLogitsLoss`
    *   作用：迫使模型关注毒性的具体表现形式，增强对毒性特征的捕获。
3.  **Identity Head (辅助任务 2)**:
    *   结构：`Linear(512 -> 9)`
    *   目标：预测文本中涉及的 9 种敏感身份 (Male, Female, Black, White, etc.)。
    *   损失函数：`BCEWithLogitsLoss`
    *   作用：显式让模型识别身份词，这在后续的 **Reweighting** 阶段至关重要——模型必须先“知道”身份存在，才能学会“解耦”身份与毒性。

## 2. 实验设置 (Experimental Setup)

所有实验均在单张 NVIDIA GPU (CUDA) 上运行。

### 2.1 训练流程 (Training Pipeline)

训练分为两个阶段，旨在平衡准确性与公平性：

1.  **Stage 1: Multi-Task Pre-training**
    *   **Objective**: 联合优化三个任务的损失。
    *   **Loss Equation**: `Loss = Loss_tox + 0.5 * Loss_sub + 0.2 * Loss_id`
    *   **Goal**: 学习通用的语义表示，并捕捉毒性和身份特征。

2.  **Stage 2: Identity-Aware Reweighting** (关键创新)
    *   **Dataset**: 使用全量数据。
    *   **Sampling Strategy**: 对“非毒性但包含身份词”的样本 (False Positive 高发区) 和“有毒但包含身份词”的样本进行特殊加权处理。
    *   **Weights**:
        *   Non-toxic + Identity (Target=0, Has_ID=1): **Weight = 2.5** (强力压制误报)
        *   Toxic + Identity (Target=1, Has_ID=1): **Weight = 1.5** (保持对有毒内容的敏感度)
    *   **Goal**: 惩罚模型利用“身份词”作为“毒性”捷径的行为，强制模型学习上下文语义。

### 2.2 超参数 (Hyperparameters)
*   **Optimizer**: AdamW
*   **Learning Rate**: 1e-5 (Stage 2), 2e-5 (Baselines)
*   **Batch Size**: 16 (Baselines), 8 (Ablations w/ accumulation)
*   **Loss Function**: BCEWithLogitsLoss (Multi-label compatiable)

## 3. 对比模型 (Baseline Models)

为了全面评估性能，我们选取了不同架构的基线模型：

| 模型代号 | 架构描述 | 核心差异 |
| :--- | :--- | :--- |
| **Baseline 1** | **BERT-CNN-BiLSTM** | 传统深度学习架构。在 BERT 输出之上叠加 CNN 提取局部特征，BiLSTM 提取序列特征。代表了 Pre-Transformer 时代的复杂特征工程思路。 |
| **Baseline 2** | **BERT-Base** | 标准 Transformer 基线。使用 `[CLS]` token 进行分类。用于衡量 DeBERTa 架构本身的优势。 |
| **Baseline 3** | **RoBERTa-Base** | 改进的 BERT。移除了 NSP 任务，使用了更大的 Batch Size 和更多的数据。通常比 BERT 强。 |
| **Baseline 4** | **DeBERTa-V1** | 第一代 DeBERTa。引入了 Disentangled Attention 机机制，但缺乏 V3 的 RTD (Replaced Token Detection) 预训练任务。 |
| **Proposed** | **DeBERTa-V3 (MTL)** | 本研究模型。引入 V3 改进 + 多任务监督 + 注意力池化 + 身份重加权。 |

## 4. 消融实验设置 (Ablation Study Variants)

为了验证各组件的有效性，我们设计了以下变体：

1.  **w/o Reweighting**
    *   **配置**: 仅进行 Stage 1 训练，不进行 Stage 2 重加权。
    *   **目的**: 验证重加权策略对消除偏见 (BPSN AUC 提升) 的贡献。

2.  **w/o MTL (Single Task)**
    *   **配置**: 移除 Subtype Head 和 Identity Head，`Loss = Loss_tox`。
    *   **目的**: 验证多任务学习是否能辅助主任务，提升泛化能力。

3.  **w/o Pooling (CLS Only)**
    *   **配置**: 移除 Attention Pooling 层，直接使用最后一层 `[CLS]` 向量进行分类。
    *   **目的**: 验证聚合所有 token 信息是否优于仅使用 `[CLS]`。
